# Hero Recipe: 쿠폰 사용 통계

사용 시나리오: 쿠폰 캠페인 종료 후 발행/사용/잔여 현황을 집계해 보고할 때.

## 1단계 — 쿠폰 목록 추출

```
backnd coupon list --json
```

offset 기반 페이지네이션 — 반환 건수가 `--limit`보다 적을 때까지 `--offset`을 증가시키며 전체 수집.

## 2단계 — 쿠폰별 사용 내역 추출

각 쿠폰 ID에 대해 사용 내역 조회:
```
backnd coupon usage --json --coupon-id <coupon-id>
```

건수가 많으면 동일하게 페이지네이션 처리.

## 3단계 — 집계

각 쿠폰에 대해:
- **발행 수**: `coupon list`의 `total_count` 또는 `issued_count` 필드
- **사용 수**: `coupon usage`의 항목 수 합산
- **잔여 수**: 발행 수 - 사용 수
- **사용률**: 사용 수 / 발행 수 × 100

## 4단계 — 결과 보고

집계 결과를 테이블 형식으로 출력:

| 쿠폰명 | 발행 | 사용 | 잔여 | 사용률 |
|--------|------|------|------|--------|
| ...    | ...  | ...  | ...  | ...%   |

## 주의사항

- `coupon usage` 응답 구조는 CLI 버전에 따라 바뀔 수 있다. `--help`로 필드명 확인.
- 대용량 캠페인(발행 수만 건)은 전체 페이지네이션 완료까지 시간이 걸린다.
  중간 진행 상황을 사용자에게 알려줄 것.
